You've got the basics down. Here are some good next steps.

## Explore the Library

The best way to learn what's possible is to see what others have built. Browse the [Library](/getting-started/library) — check out widgets, try different themes, and import spaces that interest you.

## Build Something for Yourself

Pick a small, real problem and build a widget for it:

- A **habit tracker** for your daily routines
- A **calculator** tailored to something specific in your work
- A **dashboard** for data you check every day
- A **planning tool** for an upcoming trip or project

Starting small lets you learn the workflow without getting overwhelmed. You can always add complexity later.

## Try Themes and Spaces

Customize the look of your workspace with a theme from the library, or create your own by uploading an image. Set up a space with your favorite widgets and publish it as a template for others.

## Deploy a Website

Once you've built a widget you're proud of, [deploy it as a standalone site](/getting-started/sharing#deploy-as-a-website). It gets its own URL at `yourname.app.space` and works as an independent web app anyone can visit.

## Go Deeper (For Developers)

If you want to understand how things work under the hood — or take direct control of widget code — check out the [Developer docs](/developers/overview). Topics include:

- [Local development](/developers/local-development) with GitHub and your own editor
- [How storage works](/developers/storage) and where your data lives
- [Every hook in the SDK](/developers/hooks) and when to use them
- [Deploying as a standalone site](/developers/deployment) and how storage is handled
